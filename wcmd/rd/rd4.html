WordPress Template Hierarchy System

The Template Heirarchy System

The Template Hierarchy System is how WordPress determines which template file to use for a given page. It works by matching every query string string to a query type to find which page is being requested. Then it selects the template in the order. Finally, it looks for template files with specific names in the current directory and uses the first one it finds.

The Order

WordPress reads template files in very specific order. First, it looks for a template file. Next, it looks for a specialized template that contains a page slug. After that, it looks for a specialized template with an ID. If no template is found then it uses the default page template. If page.php or default is not found then it uses singularl.php or a single posts template. Lastly, if no page templates are found it will default to using the index file.

File Naming Conventions

WordPress uses a very specific naming convention for template files. If the template is not specialized do not use a hyphen after the first word. For example, page_full-width.php is a correctly written filename. If it is a specialized template then it can have a hyphen followed by the slug or an ID.

Summary

The Template Hierarchy System is how WordPress determines which template file to use for a given page. It works by matching every query string string to a query type to find which page is being requested. The order is the way that WordPress reads the template files. Lastly, the file naming conventions are how WordPress knows which template file to use.
